📋 Contract

Universal Response Shape

Every API response follows one of exactly two shapes. No exceptions.

✅ Success

{
  "msg":     "Human readable",
  "success": true,
  "code":    200,
  "error":   null,
  "data":    { ... }
}

❌ Error

{
  "msg":     "Human readable",
  "success": false,
  "code":    422,
  "error": {
    "issue":   "Validation Failed",
    "details": [ ... ]
  },
  "data": null
}
🖥️ Database

Database Schema

4 PostgreSQL schemas. The app only reads through views — which enforce expiry and single-use constraints centrally.

authTables
user_details
Active users. PK uuid. Unique: user_id, email, phone. Soft delete via is_deleted.
pending_user_registration
Signup staging. Expires in 15 min. No FK to user_details.
security_details
hashed_password, is_2fa_enabled, is_email_verified. FK→user_details(user_id).
auth_viewsViews
user_profile
JOIN user_details + security_details. Filters is_deleted=false. Computes full_name.
pending_user_registration_vw
Filters: expires_at > now() AND is_token_used = false.
servicesTables
token_transaction_log
PK token_id (embedded in JWT). action=purpose. is_token_used for single-use.
otp_transaction_log
hashed_otp (Argon2). is_used flag. Expires 15 min.
services_viewsViews
token_transaction_log_vw
Only active: expires_at > now() AND is_token_used = false.
otp_transaction_log_vw
Only active: expires_at > now() AND is_used = false.
👤 Flows

Auth Flows

Step-by-step walkthrough of every major user journey.

1 New User Signup

API
POST /v1/auth/signup
Validates payload. Checks user status (ACTIVE→409, PENDING/NEW→continue).
DB
Insert into pending_user_registration
Pre-assigned user_id generated. SIGN_UP JWT created and stored in token_transaction_log.
EMAIL
Activation email sent (background task)
Link: /pages/auth/set-password?token=<JWT>. Expires in 15 minutes.
DB
POST /v1/auth/signup/set-password
Argon2 hash stored. Transaction: final_onboarding → setup_new_password. Token + pending row marked used.
INSERT user_details → INSERT security_details → COMMIT
DONE
Account active — 2FA enabled by default
Returns user_id + email. Redirect to login.

2 Login with OTP 2FA

API
POST /v1/auth/otp/login
Credential resolved (email / phone / user_id). Password Argon2-verified against stored hash.
DB
Fetch user_profile from auth_views
View returns active, non-deleted user. is_active checked at application layer.
OTP
4-digit OTP generated + emailed (background)
OTP Argon2-hashed and stored in otp_transaction_log. challenge_token JWT returned.
returns: { mode:"otp", challenge_token:"…" }
API
POST /v1/auth/otp/login/2fa_verify
challenge_token decoded → otp_id extracted → DB lookup → Argon2 verify. OTP marked used.
DONE
Session token issued
LOGIN_SESSION JWT in body + Authorization cookie set. Token stored in token_transaction_log. Single-use.

3 Forgot Password

API
POST /v1/auth/forgot-password
Credential resolved. User fetched from auth_views.user_profile.
DB
PASSWORD_RESET token created
JWT stored in token_transaction_log. Expiry 15 minutes.
EMAIL
Reset link sent (background task)
Always 202 — response is vague to prevent email enumeration.
DB
POST /v1/auth/reset-password
Token validated (unexpired, unused, active user). Argon2 hash updated in security_details. Token marked used.
UPDATE auth.security_details … RETURNING last_updated
DONE
Password updated — success email sent
Confirmation email with timestamp dispatched as background task.
